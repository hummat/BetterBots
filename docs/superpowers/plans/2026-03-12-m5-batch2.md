Add a helper and integrate into the existing hook:

```lua
local POXBURSTER_HUMAN_SUPPRESS_DIST = 8

local function _is_poxburster_near_any_human(pox_unit)
  local pox_pos = POSITION_LOOKUP[pox_unit]
  if not pox_pos then
    return false
  end

  local extension_manager = Managers and Managers.state and Managers.state.extension
  if not extension_manager then
    return false
  end

  local side_system = extension_manager:system("side_system")
  if not side_system then
    return false
  end

  -- Iterate player units on bot's side to find humans
  local side = side_system:get_default_player_side()
  if not side or not side.valid_human_units then
    return false
  end

  for _, human_unit in ipairs(side.valid_human_units) do
    local human_pos = POSITION_LOOKUP[human_unit]
    if human_pos and Vector3.distance(pox_pos, human_pos) < POXBURSTER_HUMAN_SUPPRESS_DIST then
      return true
    end
  end

  return false
end
```

### Task 4: Ping system redesign — target tracking + distance escalation (#16)

**Branch:** `feat/16-ping-target-tracking`

**Context:** Current implementation uses a 2s fixed cooldown. This causes tag-flipping between multiple alive elites. The redesign (from issue comments) replaces the cooldown with per-bot target tracking and distance escalation.

**Files:**
- Modify: `scripts/mods/BetterBots/ping_system.lua` — replace cooldown with target tracking
- Modify: `tests/ping_system_spec.lua` — update tests for new behavior

**Design (from issue #16 comments):**

State per bot (weak-keyed):
```lua
_last_tagged_by_bot[unit] = { target = unit, dist_sq = number }
```

**Suppress re-tagging** while last-tagged target is:
- Still alive (`Unit.alive`)
- Still tagged (`tag_id() ~= nil`) OR still in any perception slot

**Allow retag** when:
1. Old target died/despawned
2. Old target's tag expired AND it left all perception slots
3. Distance escalation: `new_dist_sq < 0.5 * old_dist_sq` (new threat significantly closer)

**What gets removed:** `PING_COOLDOWN_S`, `_last_ping_t_by_bot`
**What gets added:** `_last_tagged_by_bot`, `_is_in_any_perception_slot()`, distance escalation check

Replace `_last_ping_t_by_bot` and `PING_COOLDOWN_S` with:

```lua
local _last_tagged_by_bot = setmetatable({}, { __mode = "k" })
-- Each entry: { target = unit, dist_sq = number }

local ESCALATION_RATIO = 0.5 -- new target must be this fraction of old distance

local function _is_in_any_perception_slot(perception, target)
  for i = 1, #PING_SLOTS do
    if perception[PING_SLOTS[i]] == target then
      return true
    end
  end
  return false
end

local function _should_suppress_retag(last, perception, new_dist_sq)
  if not last or not last.target then
    return false
  end
  if not Unit.alive(last.target) then
    return false -- old target dead, allow retag
  end
  local tag_ext = ScriptUnit.has_extension(last.target, "smart_tag_system")
  local still_tagged = tag_ext and tag_ext:tag_id()
  local still_perceived = _is_in_any_perception_slot(perception, last.target)
  if not still_tagged and not still_perceived then
    return false -- forgotten, allow retag
  end
  -- Distance escalation: allow if new target is significantly closer
  if new_dist_sq and last.dist_sq and new_dist_sq < ESCALATION_RATIO * last.dist_sq then
    return false
  end
  return true -- suppress
end
```

Update `M.update()` to use `_should_suppress_retag` instead of the cooldown check. After a successful tag, store `_last_tagged_by_bot[unit] = { target = target_unit, dist_sq = dist_sq }`.

Compute `dist_sq` for the best candidate using `Unit.world_position` + `Vector3.distance_squared`.
